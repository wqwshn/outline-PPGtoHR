# 短窗口运动状态分类器改进设计

## 1. 问题分析

当前 8 秒窗口分类器在波比跳运动分解任务上表现不佳：

- RF GroupKFold(5) accuracy: 0.780 +/- 0.390（方差极大）
- 73% 的波比跳窗口被预测为俯卧撑，几乎不预测手臂弯举和跳绳
- 平均最大概率仅 0.684，分类器对波比跳窗口的判别不确信

**根因**：波比跳一个完整周期约 2-4 秒（下蹲 -> 后踢 -> 俯卧撑 -> 收腿 -> 跳跃），8 秒窗口内混合 4-6 个动作阶段，窗口特征被占比最大的动作主导，分类退化为"哪个动作在这个窗口里停留时间最长"。

## 2. 改进策略

将分类窗口从 8 秒缩短至 2 秒，匹配波比跳单动作阶段的物理持续时间。短窗口内的信号更"纯净"，分类后通过聚合回 8 秒对下游透明。

### 窗口与特征对比

| 参数 | 原方案 | 改进方案 |
|------|--------|----------|
| 窗口长度 | 8s | 2s |
| 步长 | 1s | 0.5s |
| 采样点/窗口 | 800 | 200 |
| 特征维度 | 75 | ~50-60 |

### 特征调整

- **时域特征保留**（鲁棒）：均值、标准差、极差、过零率、能量
- **频域特征降权或裁剪**：2 秒窗口频率分辨率仅 0.5Hz，谱熵和谱平坦度估计质量下降
- **新增短窗口特有特征**：信号偏度（skewness）、峰度（kurtosis）-- 反映动作冲击性差异

## 3. 分类流程

```
原始 MIMU 信号 (100Hz)
     |
     v
2s 窗口, 0.5s 步长 -> 特征提取 (~50-60维)
     |
     v
RF 分类器 -> 每个短窗口的 4 类概率 P_i(short_t)
     |
     v
8s 聚合窗口 (覆盖范围内的所有短窗口)
     |
     v
加权平均 -> 8s 粒度概率 P_i(8s_t)
```

聚合方式：对每个 8 秒对齐点，取 [t, t+8s) 范围内的所有短窗口概率做均值。可选高斯核加权使靠近窗口中心的短窗口权重更高。

## 4. 训练与评估

### 训练数据

- 简单运动 CSV 按文件前缀标注标签
- 2s 窗口 + 0.5s 步长 -> 每类样本从 49-132 增加到约 200-530
- push_up 从 49 -> ~200，缓解类别不平衡

### 波比跳推理

- 两个 bobi 文件运动段合并处理，不需要分文件
- 静息段标记但不参与分类评估

### 评估指标（优先级）

1. **预测分布合理性**：波比跳窗口不再是 73% 单一类别，四类预测比例更均衡
2. **CV 准确率 + 稳定性**：GroupKFold(5) 准确率目标 0.85+，标准差 < 0.15

## 5. HMM 可选增强（第二步）

如果 RF 短窗口分类仍有帧间抖动，追加 HMM 平滑层：

- 训练：从简单运动短窗口标签序列学习转移概率
- 推理：Viterbi 解码得到最可能的状态序列
- 输出：平滑后概率聚合到 8 秒

先看纯 RF 短窗口效果，不够再加。

## 7. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 窗口策略 | 短窗口 + 聚合回 8s | 匹配物理过程，下游零改动 |
| 全局缩短窗口 | 否 | 心率求解器依赖 8s 窗口的频谱分析 |
| 分类器选型 | RF 先行 + HMM 可选 | 数据量小(<1700)，RF 鲁棒；HMM 渐进式加入 |
| 波比跳数据处理 | 两文件合并 | 提升数据量，测试条件一致 |
| Notebook 组织 | 新建 01b/02b | 保留原版做基线对比 |
