# bobi1 各预热长度贝叶斯优化实验

## 1. 动机

修订版实验发现：优化参数 + 预热几乎无增量收益。但这存在一个方法论局限——优化参数是在 **prewarm_seconds=0** 的条件下搜索得到的。

一个未被验证的假设是：如果优化器知道预热可用，它可能会选择不同的参数组合。例如：

| 参数 | 无预热最优 | 有预热可能的选择 | 理由 |
|------|-----------|-----------------|------|
| `fs_target` | 50 Hz (bobi1 HF) | 25 Hz | 预热提供了额外样本，降低了对高采样率的需求 |
| `lms_mu_base` | 0.012 | 0.008-0.01 | 预热缓解了收敛压力，可以用更保守的步长 |
| `max_order` | 12-16 | 8-12 | 预热让低阶滤波器也能充分收敛 |
| `smooth_win_len` | 9 | 5-7 | 滤波输出更稳定，减少平滑需求 |

本实验对每个预热长度独立运行贝叶斯优化，观察参数选择是否随预热长度系统性变化。

## 2. 实验设计

### 2.1 范围

| 维度 | 取值 |
|------|------|
| 样本 | `multi_bobi1`（唯一） |
| 参考方案 | `("HF",)`, `("ACC",)` |
| 预热长度 | [0, 2, 4, 6, 8, 10, 12] |
| 预热策略 | `motion_and_recovery` |

### 2.2 贝叶斯优化配置

```python
V2BayesConfig(
    max_iterations=75,   # 每次 repeat 的 trial 数
    num_seed_points=10,  # 初始随机种子点数
    num_repeats=3,       # 独立重复次数
    random_state=42,
)
```

### 2.3 搜索空间

| 参数 | 候选值 |
|------|--------|
| `fs_target` | [25, 50] |
| `max_order` | [8, 12, 16] |
| `lms_mu_base` | [0.008, 0.01, 0.012] |
| `smooth_win_len` | [5, 7, 9] |
| `spec_penalty_width` | [0.1, 0.2, 0.3] |
| `spec_penalty_weight` | [0.1, 0.2, 0.4] |
| `hr_range_hz` | [20, 25, 30, 35] / 60 |
| `slew_limit_bpm` | [8, 10, 12, 14] |
| `slew_step_bpm` | [5, 7, 9] |

### 2.4 运行量

- 2 ref × 7 prewarm = **14 次优化**
- 每次优化：10 + 75 = 85 trials（含种子点），重复 3 次 = 255 次 `solve_v2` 调用
- 总计：14 × 255 = **3570 次求解**
- 预估耗时：每次 solve_v2 ~0.5s → 约 30 分钟

## 3. 分析计划

### 3.1 主要分析

| 问题 | 分析方法 |
|------|---------|
| 最优参数是否随预热长度变化？ | 对比各预热长度下的 best_params |
| 预热是否降低对 fs_target=50 的偏好？ | 观察 fs_target 选择频率 vs 预热长度 |
| 预热是否让更低 mu/M 变得可行？ | 观察 lms_mu_base / max_order 趋势 |
| 预热长度与最优 AAE 的关系 | 绘制 prewarm vs best_error 曲线 |

### 3.2 判断标准

若预热有效，预期观察到：
- 预热长度 > 0 时，`fs_target=25` 被选中次数增加
- 预热长度 > 0 时，`lms_mu_base` 和 `max_order` 的中位数下降
- 较长预热长度的 best_error 优于或等于 shorter 预热

若预热无效：
- 所有预热长度的最优参数分布相似
- best_error 不随预热长度单调改善

- 结论：预热是否改变优化器行为
